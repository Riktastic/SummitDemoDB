Summit Demo DB

ERP dataset based on real system architecture for data visualization and analysis

About Summit Electronics

The dataset represents operations of Summit Electronics, a fictional mid-sized consumer electronics and home goods retailer founded in 2018. Headquartered in Amsterdam, Netherlands, the company operates through e-commerce, 45 physical retail locations across Europe and North America, and B2B sales channels.

Key Company Details:

  • Founded: 2018 in Amsterdam, Netherlands
  • Revenue: €50M annually (2024)
  • Operations: 45 retail locations across Netherlands, Germany, Belgium, France, UK, US, and Canada
  • Product Categories: Audio/Video, Smart Home, Computing, Mobile Devices, Home Appliances
  • Business Model: E-commerce (65%), Retail Stores (25%), B2B Sales (10%)

The dataset captures a snapshot of Summit Electronics' operations from January 2023 to December 2024, including seasonal demand patterns, regional market differences, supply chain complexity, and customer segmentation challenges that make the data meaningful for analysis.

Overview

This project generates a fully interconnected ERP dataset using Odoo 18, suitable for data visualization, analysis, and BI exercises. The dataset includes customers, suppliers, products, sales and purchase orders, deliveries, invoices, bills, payments, refunds, and credit notes.

🎯 Varied Data

Regional deals, seasonal patterns, and natural causality

📊 Multiple Formats

CSV, SQLite, PostgreSQL, MSSQL, MariaDB

🔗 Fully Connected

All tables properly linked with referential integrity

📈 Time-Series Ready

Date distribution over 2 years

Quick Start

Generate the complete dataset with a single command:

./run-all.sh

This will:

  • Initialize Odoo with required modules
  • Generate ERP records (200 customers, 100 suppliers, 300 products, 500 sale orders, 300 purchase orders)
  • Export all tables to CSV format

To create a GitHub release package with all export formats:

./run-all.sh --create-release

Data Realism Features

The generated data includes several realism enhancements:

  • Regional Deals: Promo codes restricted by region (e.g., Europe-only deals)
  • Seasonal Patterns: Holiday-based ordering bias (Christmas, Black Friday, New Year)
  • Product Seasonality: Different product categories preferred by month
  • Skewed Demand: 20% of customers generate 60% of orders
  • Promoted Products: 5% of products have 15% higher demand
  • Date Distribution: Order dates spread over 2 years with proper date sequencing
  • Mixed Statuses: Orders have confirmed, cancelled, and draft states

Export Formats

The data can be exported in multiple formats:

  • CSV: Individual files for each table in csv_export/
  • SQLite: Single database file with all tables
  • PostgreSQL: Native SQL dump for PostgreSQL
  • MSSQL: Converted SQL for Microsoft SQL Server
  • MariaDB: Converted SQL for MariaDB/MySQL

Scale Configuration

Default scale (configurable via environment variables):

  • 200 customers
  • 100 suppliers
  • 300 products
  • 500 sale orders
  • 300 purchase orders

Example:

export GEN_SALE_ORDERS="5000"
export GEN_PRODUCTS="2000"
./run-all.sh
